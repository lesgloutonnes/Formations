💬 Règle importante avec l'IA

Dès que vous passez à une autre macro, un autre exercice ou une autre demande, ouvrez une nouvelle conversation dans votre outil IA. C'est plus sûr : l'IA ne mélangera pas le code, les noms de feuilles ou les consignes des exercices précédents.

Vous pouvez garder la même conversation uniquement pour corriger ou améliorer la macro en cours.

Prérequis : Modules 1 & 2 · Excel + Edge + IA autorisée

✏️ Exercice 1
Préparer le poste + BonjourCHC
📁 EXERCICES\Ex01_Environnement.xlsx ⏱️ Environ 15 minutes

Ouvrez Ex01_Environnement.xlsx dans le dossier EXERCICES. Feuille Demarrage pour le rappel des étapes.

Exercice — à réaliser maintenant

  1. Activez l'onglet Développeur si ce n'est pas déjà fait.
  2. Appuyez sur Alt+F11 pour ouvrir l'éditeur VBA, parcourez la fenêtre, puis fermez-la.
  3. Placez Excel et Edge avec votre outil IA côte à côte à l'écran.
  4. Dans l'IA, demandez par exemple : « Crée une macro VBA Excel nommée BonjourCHC qui affiche une MsgBox : Bonjour, équipe [nom de mon service] ! »
  5. Collez le code : Alt+F11 → Insertion → Module → collez → Alt+F8 → Exécuter → vérifiez que la MsgBox s'affiche.
  6. Enregistrez d'abord en .xlsm (Classeur Excel avec macros).
  7. Demandez ensuite à l'IA : « Ma macro BonjourCHC affiche une MsgBox. Fais en sorte qu'elle s'affiche automatiquement à chaque ouverture du fichier. Voici mon code : [coller]. »
  8. Collez le code proposé dans ThisWorkbook (double-clic sur « ThisWorkbook » dans l'arborescence VBA, pas dans un module standard).
  9. Fermez Excel, rouvrez le .xlsm : la MsgBox doit apparaître dès l'ouverture (acceptez les macros si Excel le demande).
✅ Correction 1
BonjourCHC
  1. Vérifier l'onglet Développeur visible.
  2. Module standard — macro BonjourCHC : 
    Sub BonjourCHC()
    MsgBox "Bonjour, équipe Urgences !", vbInformation, "CHC"
End Sub
    (adapter le nom du service)
  3. Alt+F8 → BonjourCHC → Exécuter : la MsgBox s'affiche.
  4. Enregistrer en .xlsm.
  5. Objet ThisWorkbook — événement à l'ouverture : 
    Private Sub Workbook_Open()
    Call BonjourCHC
End Sub
    Le code va dans ThisWorkbook, pas dans Module1.
  6. Fermer et rouvrir le fichier : la MsgBox s'affiche sans passer par Alt+F8.
  7. Si rien ne se passe : macros activées pour ce fichier, et code bien dans ThisWorkbook (feuille « Classeur » dans l'arborescence VBA).

✏️ Exercice 2
FormaterTableau
📁 EXERCICES\Ex02_FormaterTableau.xlsx ⏱️ Environ 15 minutes

Ouvrez Ex02_FormaterTableau.xlsx, feuille Import_Brut. Plage à formater : A1:D9 (données sans mise en forme).

Exercice — à réaliser maintenant

  1. Enregistrez une macro nommée FormaterTableau : sélectionnez A1:D9, appliquez en-têtes en gras, couleur verte, bordures, autofiltre, puis arrêtez l'enregistrement.
  2. Testez avec Alt+F8 : la plage doit être mise en forme d'un seul coup.
  3. Ouvrez Alt+F11 et copiez tout le code de la macro.
  4. Dans l'IA, demandez d'améliorer le code (et aussi de donner un look encore plus 'professionnel'), remplacez le code, retestez.
  5. Enregistrez en .xlsm.
✅ Correction 2
FormaterTableau
  1. Macro enregistrée sur Feuil1 ou Import_Brut selon le nom de feuille active.
  2. Résultat attendu : en-têtes blancs sur fond vert, bordures, flèches de filtre.
  3. Si erreur « méthode impossible » : préciser à l'IA le nom exact de la feuille.
  4. Bonus : demander « figer la ligne 1 » dans le même prompt.

💡 Modèle de prompt (exercice 3)

« Crée une macro VBA Excel. Feuille : Liste_Statuts. Colonne C = Statut. Parcours les lignes 2 à 13 : si la cellule vaut Inactif, colorie toute la ligne en gris clair. Nomme la procédure ColorerInactifs (c'est le nom qui apparaîtra dans Alt+F8). »

En VBA, le nom de la macro est la ligne Sub ColorerInactifs() — pas un commentaire à la fin du prompt.

Débogage : « Erreur [texte] ligne [N]. Objectif : … Code : [coller]. Corrige. » — puis « Explique ligne par ligne ».

✏️ Exercice 3
ColorerInactifs
📁 EXERCICES\Ex03_Statuts_CHC.xlsx ⏱️ Environ 18 min (+ 10 min pour aller plus loin)

Ouvrez Ex03_Statuts_CHC.xlsx, feuille Liste_Statuts. Colonnes Nom, Service, Statut — repérez les lignes Inactif.

Exercice — à réaliser maintenant

  1. Utilisez le prompt de la slide théorie (adaptez le nom de feuille si besoin).
  2. Collez le code VBA dans un module, puis Alt+F8 → Exécuter.
  3. Vérifiez que chaque ligne dont le statut est Inactif est colorée en gris.
  4. Demandez à l'IA d'expliquer le code (For, If, Cells).
  5. Enregistrez en .xlsm.

🚀 Pour aller plus loin (même conversation IA que la version de base)

Quand la macro de base fonctionne, copiez votre code dans l'IA et utilisez ce prompt :

« J'ai une macro ColorerInactifs sur la feuille Liste_Statuts (colonne C = Statut). Améliore le fichier sans renommer la macro :

1) Dernière ligne automatique — la macro parcourt toutes les lignes avec données, sans numéro de ligne fixe.

2) Liste déroulante — en colonne C à partir de la ligne 2, choix uniquement Actif ou Inactif.

3) Mise à jour dynamique — dès qu'on change le statut dans la liste, la ligne entière passe en gris si Inactif, ou retrouve l'affichage normal si Actif, sans relancer Alt+F8.

Utilise si besoin Worksheet_Change sur la feuille Liste_Statuts. Voici mon code actuel : [coller]. »

  1. Remplacez ou complétez le code (souvent code dans le module + événement dans la feuille).
  2. Lancez une fois ColorerInactifs (Alt+F8) pour appliquer listes et couleurs initiales.
  3. Testez : changez un statut en Inactif, puis en Actif — la ligne doit réagir tout de suite.
  4. Enregistrez le .xlsm.
✅ Correction 3
ColorerInactifs
  1. Version de base : boucle For i = 2 To … · If Cells(i,3) = "Inactif" Then · fond gris sur la ligne.
  2. 6 lignes « Inactif » grisées dans le fichier fourni. Erreur fréquente : mauvaise colonne (Statut = C).
  3. Aller plus loin : End(xlUp) ou CurrentRegion pour la dernière ligne · validation de données Actif/Inactif en colonne C · Worksheet_Change pour recolorer à chaque modification.
  4. Si la feuille ne réagit pas au changement : vérifier que le code d'événement est bien dans l'objet feuille Liste_Statuts, pas dans un module standard.

Tableau de bord
PARTIE 4
3 boutons — un collègue clique sans connaître VBA

Vous allez relier trois macros à des boutons pour qu'un collègue puisse les lancer sans ouvrir le VBA.

Nouvelle conversation IA avant cet exercice (vous pourrez la réutiliser pour les deux autres macros du tableau de bord).

📚 Théorie
Boutons sur la feuille

🔧 5 gestes

  • Développeur → Mode Création
  • Insérer → Bouton (Formulaires) → dessiner
  • Assigner la macro → renommer le libellé
  • Répéter pour 3 macros
  • Désactiver Mode Création → tester les clics

Libellés clairs : Formater, Effacer, Ajouter ligne — pas « Bouton1 ».

✏️ Exercice 4
Tableau de bord stocks
📁 EXERCICES\Ex04_TableauDeBord.xlsx ⏱️ Environ 20 minutes

Ouvrez Ex04_TableauDeBord.xlsx : feuille Donnees (A1:C5) et feuille Tableau_de_bord pour placer les boutons.

Exercice — à réaliser maintenant

  1. Macro FormaterTableau : réutilisez celle de l'exercice 2 ou réenregistrez sur la plage Donnees !A1:C5.
  2. Avec l'IA, créez EffacerStocks pour vider les cellules A2:C5 de la feuille Donnees.
  3. Avec l'IA, créez AjouterLigne pour ajouter une ligne vide sous les données.
  4. Sur la feuille Tableau_de_bord, ajoutez trois boutons reliés aux trois macros.
  5. Faites tester par un voisin, puis enregistrez en .xlsm.
✅ Correction 4
Tableau de bord
  1. 3 macros listées dans Alt+F8.
  2. Boutons sur la feuille dashboard, pas sur Donnees (plus lisible).
  3. Test : Formater → Effacer → Ajouter ligne → re-Formater.
